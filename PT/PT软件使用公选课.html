快速计算

一 快速计算的原则

  1. 降低计算复杂度
  2. 降低内存和输出设备的访问频率

1.1 计算复杂度

  1. 单位：次基本运算
  2. 基本运算： +−×÷sin⁡cos⁡tan⁡ln⁡∣□∣⋯+-\times\div \sin\cos\tan\ln |\Box| \cdots+−×÷sincostanln∣□∣⋯
  3. 函数和复杂运算：基本运算的线性和阶组合

1.2 空间复杂度

  1. 单位：位
  2. 函数申请变量（类比指针）

1.3 时间开销

  1. 花费单位时间：
    1. 一次内存的读写操作
    2. 一次基本运算
  2. 输出的时间开销：
    1. 多次内存的读写操作
    2. 可能：
      1. GPU等显示设备内存的读写
      2. 转译

1.4 协同和集群计算

graph LR; A(CPU 0)-->B(CPU 1); A-->C(CPU 2); A-->D(CPU 3)
  • 任务分配和独立计算
  • 可减轻大型循环体和大型数组索引的单核计算压力
  • 提高系统利用效率
  • 缺点：
    • 单核计算效率降低
    • 很多自定义函数不能利用集群计算

1.5 计算加速卡（图形显示卡）计算

  • NVIDIA显卡CUDA和TF计算核心
  • 显示卡集群计算
graph LR; A(CPU)-->B(GPU 0); A-->C(GPU 1); A-->D(GPU 2); A-->E(GPU 3)
  • 瓶颈
    • PCIE协议的传输效率
    • CPU的调度效率
    • VRAM访问效率

2.1 Matlab Pool

2.1.1 使用方法

版本Matlab<2016

```matlab matlabpool open; parfor ... end matlabpool close; ```

版本Matlab>2016

```matlab pool open; parfor ... end pool close; ```

2.1.2 原理

把循环体中接下来要进入循环的内容分配和计算

2.1.3 注意点

  1. 如果单个循环中计算压力不大，可能会使计算效率降低
  2. parfor不能嵌套
    ⇒\Rightarrow⇒
  3. 多层循环在最外层利用分流计算
  4. 减少函数的自身反复调用造成的嵌套

2.2 代码减负

  1. 重复利用已经生成好的变量
  2. 数组预先初始化，减少动态申请空间的压力
    x=zeros(500,1)
  3. 减少循环体中的局部变量（生存期前后要申请和销毁，费时间）
  4. 减少函数嵌套循环
  5. 减少输出
  6. 利用基础运算方法代替内置函数
  7. 减少符号运算
  8. 能直接打就不必建立函数
  9. 简化求解：
    1. 线性近似
    2. 只完成题目要求

2.3 计算减负

  1. 避免对很小的数字计算(代换)
  2. 计算域有类似的规模(可以利用同一种变量申请)
  3. 减小步长和计算精度(精细度和容错)
    ⇒\Rightarrow⇒
    1. 多级计算
    2. 人工干预
  4. 降低计算的自由度

2.4 意义

1.更快
2.计算准确

  8. 分步人为干预
